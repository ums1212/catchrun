# PR Release v0.0.8 🚀 (v0.0.1 이후 변경 사항 정리)

v0.0.1 릴리즈 이후 진행된 주요 기능 추가, 시스템 안정화 및 UI/UX 개선 사항을 정리한 PR입니다.

## 📋 주요 변경 사항

### 1. 핵심 기능 및 게임 로직 개선 (Core Gameplay)
- **NFC 탈옥 시스템 강화**: 탈옥 키 사용 시 70점 감점 패널티 로직을 추가하여 게임 밸런스를 조정했습니다.
- **서버 시간 기반 동기화**: 모든 디바이스에서 동일한 타이머를 유지하기 위해 서버 타임스탬프 기반 동기화 로직을 적용했습니다.
- **게임 종료 로직 고도화**: 타이머 종료 시 모든 참여자의 화면에서 게임 결과가 정확히 처리되도록 보완했습니다.
- **감옥 시스템 업데이트**: 감옥 화면에서도 타이머 확인 및 게임 종료 처리가 가능하도록 기능을 추가했습니다.

### 2. UI/UX 및 편의성 강화 (UI/UX Enhancements)
- **디자인 시스템 정비**: 다이얼로그 텍스트 스타일(노란 밑줄 이슈 등)을 수정하고 배경 이미지 복구 작업을 진행했습니다.
- **네트워크 에러 핸들링**: 전역 네트워크 연결 체크를 도입하여 연결 유실 시 사용자에게 알림 다이얼로그를 표시합니다.
- **뒤로가기 방지 로직**: 홈 화면 및 게임 플레이 중 실수로 앱이 종료되지 않도록 확인 다이얼로그(PopScope)를 구현했습니다.
- **아바타 랜덤 생성**: 사용자 프로필 설정에서 랜덤 아바타 생성 및 저장 기능을 추가했습니다.

### 3. 안정성 및 버그 수정 (Standardization & Fixes)
- **QR 스캔 최적화**: 연속 스캔 방지 및 에러 발생 시 다이얼로그 처리를 통해 스캔 경험을 개선했습니다.
- **라우팅 에러 해결**: `RouteObserver` 중복 등록 이슈를 해결하여 화면 전환 안정성을 높였습니다.
- **익명 계정 관리**: 로그아웃 시 익명 계정 삭제 로직을 추가하여 클린한 사용자 관리를 지원합니다.
- **Lobby 기능 보완**: 방 나가기 로직, 공유 기능(SharePlus) 최신화 및 시스템 UI 폴리시를 적용했습니다.

## 🛠 기술적 결정 사항
- **Firestore Security Rules**: 게임 종료 시 클라이언트 트리거와 서버 검증을 병합하여 트래픽 과부하를 방지했습니다.
- **Global Error Handling**: `ConnectivityService`를 통한 중앙 집중식 네트워크 상태 관리 모델을 채택했습니다.

## ✅ 검증 결과
- 서비스 시나리오(로그인 -> 로비 -> 게임 -> 결과) 전 과정 테스트 완료
- 안드로이드/iOS 크로스 플랫폼 빌드 확인
- 실시간 DB 동기화 및 타이머 오차 범위 검증
